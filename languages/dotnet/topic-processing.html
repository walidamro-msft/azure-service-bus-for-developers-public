Topics and subscriptions in .NET

Publish events, process independent subscription copies, and manage SQL and correlation rules with Azure.Messaging.ServiceBus.

C# / .NET 8+Azure.Messaging.ServiceBusPublish, subscribe, and filter

Package, identity, and topology

Install Azure.Messaging.ServiceBus and Azure.Identity. Reuse one ServiceBusClient per namespace. Grant publishers Azure Service Bus Data Sender, subscribers Azure Service Bus Data Receiver, and topology automation Azure Service Bus Data Owner only when it must manage rules.

dotnet add package Azure.Messaging.ServiceBus
dotnet add package Azure.Identity
using Azure.Identity;
using Azure.Messaging.ServiceBus;
using Azure.Messaging.ServiceBus.Administration;

string fqns = Environment.GetEnvironmentVariable("SERVICEBUS_FQNS")
    ?? throw new InvalidOperationException("Set SERVICEBUS_FQNS.");

await using var client = new ServiceBusClient(fqns, new DefaultAzureCredential());
Fan-out boundary: a topic receives the publisher's message once. Each matching subscription owns a durable copy with independent locks, delivery count, settlement, and dead-letter state.

Publish to a topic

A topic sender uses the same ServiceBusSender and ServiceBusMessage types as a queue sender. Put fields needed by subscription rules in ApplicationProperties; rules cannot inspect JSON body fields.

await using ServiceBusSender sender = client.CreateSender("order-events");

var message = new ServiceBusMessage(BinaryData.FromObjectAsJson(new
{
    orderId = "ORD-123",
    total = 149.50
}))
{
    MessageId = "order-created:ORD-123",
    CorrelationId = "order:ORD-123",
    Subject = "OrderCreated.v1",
    ContentType = "application/json"
};
message.ApplicationProperties["region"] = "us";
message.ApplicationProperties["priority"] = 3;

await sender.SendMessageAsync(message);

Reuse the same MessageId when retrying the same logical publish. Every matching subscription copy preserves that publisher-assigned identity, so each subscriber can enforce idempotency independently.

Process one subscription

Create a processor with both the topic and subscription names. One processor instance targets one subscription path; deploy separate processors or worker roles when downstream capabilities scale independently.

await using ServiceBusProcessor processor = client.CreateProcessor(
    "order-events",
    "inventory",
    new ServiceBusProcessorOptions
    {
        AutoCompleteMessages = false,
        MaxConcurrentCalls = 4,
        MaxAutoLockRenewalDuration = TimeSpan.FromMinutes(5)
    });

processor.ProcessMessageAsync += async args =>
{
    try
    {
        await ReserveInventoryAsync(args.Message.Body);
        await args.CompleteMessageAsync(args.Message);
    }
    catch (InvalidOrderException exception)
    {
        await args.DeadLetterMessageAsync(args.Message, "InvalidOrder", exception.Message);
    }
    catch
    {
        await args.AbandonMessageAsync(args.Message);
    }
};
processor.ProcessErrorAsync += args =>
{
    Console.Error.WriteLine($"{args.EntityPath}: {args.Exception}");
    return Task.CompletedTask;
};

await processor.StartProcessingAsync();

For pull-based handling, call client.CreateReceiver(topicName, subscriptionName). Complete, abandon, defer, dead-letter, and lock-renewal behavior is identical to queue receiving, but settlement affects only this subscription's copy.

Manage subscription rules

Use ServiceBusAdministrationClient for rules. A new subscription includes $Default, which matches every message. Delete it when the subscription should receive only explicitly matched events.

var admin = new ServiceBusAdministrationClient(fqns, new DefaultAzureCredential());

await admin.DeleteRuleAsync("order-events", "inventory", RuleProperties.DefaultRuleName);

await admin.CreateRuleAsync(
    "order-events",
    "inventory",
    new CreateRuleOptions("UsOrders", new SqlRuleFilter("region = 'us' AND priority >= 2")));

await admin.CreateRuleAsync(
    "order-events",
    "billing",
    new CreateRuleOptions("OrderCreated", new CorrelationRuleFilter
    {
        Subject = "OrderCreated.v1"
    }));

SqlRuleFilter

Use forExpressions and comparisons
ReadsSystem and application properties
Choose it when

A rule combines fields, ranges, Boolean logic, or null checks. Keep expressions small and treat property names, casing, and data types as part of the event contract.

CorrelationRuleFilter

Use forExact matches
ReadsSelected system and application properties
Choose it when

The subscription matches exact values such as subject, correlation ID, or a compact property set. It is optimized for equality matching.

Operational design

  • Use a distinct subscription for every capability that needs its own durable event copy.
  • Use competing processors on the same subscription only when instances should share that subscription's work.
  • Monitor active, dead-letter, and oldest-message age per subscription rather than only at the topic.
  • Keep subscription processors long-lived and stop them gracefully before disposing the shared client.
  • Use SubQueue.DeadLetter with the topic and subscription receiver overload for focused recovery.
