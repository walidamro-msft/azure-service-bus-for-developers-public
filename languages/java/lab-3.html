Lab 3 - Preserve each order's event sequence

Provision a session-enabled queue, send interleaved workflows, and process each order in FIFO order.

60 minutesTwo Java appsSessions

Case study: ordered fulfilment workflows

Payment, allocation, and dispatch events for many orders share one queue. Different orders can run concurrently, but each order must preserve its own sequence. A Service Bus session groups messages under the order ID.

flowchart LR
 S[Java workflow sender] -- SessionId = order ID --> Q[order-sessions queue]
 Q -- one locked session --> W1[Processor worker]
 Q -- another locked session --> W2[Processor worker]

1. Provision a session-enabled queue

Reuse the namespace from Lab 1. Sessions cannot be enabled after a queue is created, so session support must be enabled during creation.

Option A: Azure portal

  1. In the Azure portal, open the Service Bus namespace you created in Lab 1.
  2. Under Entities, select Queues, then select + Queue.
  3. Set the queue name to order-sessions, enable Sessions, and set Max delivery count to 5.
  4. Confirm that sessions are enabled before you select Create. Sessions cannot be enabled after the queue is created.
  5. Set the namespace hostname copied in Lab 1 as SERVICEBUS_FQNS in every terminal used by this lab.

Option B: Azure CLI

az servicebus queue create --resource-group $resourceGroup --namespace-name $namespace --name order-sessions --enable-session true --max-delivery-count 5
$env:SERVICEBUS_FQNS = "$namespace.servicebus.windows.net"
Note: Complete either Option A or Option B, not both.

3. Send two interleaved order workflows

Replace workflow-sender/src/main/java/com/contoso/workflow/App.java with:

package com.contoso.workflow;
import com.azure.identity.DefaultAzureCredentialBuilder;
import com.azure.messaging.servicebus.*;

public class App {
  public static void main(String[] args) {
    String fqns = System.getenv("SERVICEBUS_FQNS");
    if (fqns == null) throw new IllegalStateException("Set SERVICEBUS_FQNS first.");
    try (ServiceBusSenderClient sender = new ServiceBusClientBuilder()
        .fullyQualifiedNamespace(fqns).credential(new DefaultAzureCredentialBuilder().build())
        .sender().queueName("order-sessions").buildClient()) {
      String[] orderIds = {"ORD-101", "ORD-202"};
      String[] states = {"PaymentAccepted", "StockAllocated", "Dispatched"};
      for (String state : states) for (String orderId : orderIds) {
        sender.sendMessage(new ServiceBusMessage(state).setSessionId(orderId)
            .setMessageId(orderId + "-" + state).setSubject(state));
        System.out.println("Sent " + orderId + ": " + state);
      }
    }
  }
}

4. Process sessions concurrently

Replace workflow-processor/src/main/java/com/contoso/workflow/App.java with:

package com.contoso.workflow;
import com.azure.identity.DefaultAzureCredentialBuilder;
import com.azure.messaging.servicebus.*;
import java.util.Scanner;

public class App {
  public static void main(String[] args) {
    String fqns = System.getenv("SERVICEBUS_FQNS");
    if (fqns == null) throw new IllegalStateException("Set SERVICEBUS_FQNS first.");
    ServiceBusProcessorClient processor = new ServiceBusClientBuilder()
        .fullyQualifiedNamespace(fqns).credential(new DefaultAzureCredentialBuilder().build())
        .sessionProcessor().queueName("order-sessions").disableAutoComplete()
        .maxConcurrentSessions(2).maxConcurrentCalls(1)
        .processMessage(context -> {
          System.out.printf("%s: %s%n", context.getMessage().getSessionId(), context.getMessage().getBody());
          try { Thread.sleep(300); } catch (InterruptedException exception) { Thread.currentThread().interrupt(); }
          context.complete();
        })
        .processError(context -> System.err.println(context.getException().getMessage()))
        .buildProcessorClient();
    processor.start();
    System.out.println("Session processor is running. Press Enter to stop.");
    new Scanner(System.in).nextLine();
    processor.close();
  }
}
