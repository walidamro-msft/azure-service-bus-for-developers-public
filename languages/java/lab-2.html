Lab 2 - Publish one event to two subscribers

Create a topic, build a Java publisher, and run one reusable subscriber for two independent subscriptions.

60 minutesThree processesTopics

Case study: notify inventory and customer care

Checkout sends one OrderCreated event. Service Bus copies it into two durable subscriptions so inventory and customer care receive the event independently.

flowchart LR
   P[Java publisher] -- one event --> T[order-events topic]
   T -- copy --> I[inventory subscription]
   T -- copy --> C[customer-care subscription]
   I -- receive --> SI[Subscriber: inventory]
   C -- receive --> SC[Subscriber: customer-care]
Important: Run each receiver against a different subscription. Two receivers using the same subscription are competing consumers and split messages.

1. Provision the topic and subscriptions

Reuse the namespace and variables from Lab 1.

Option A: Azure portal

  1. In the Azure portal, open the Service Bus namespace you created in Lab 1.
  2. Under Entities, select Topics, then select + Topic.
  3. Set the topic name to order-events, keep the other defaults, and select Create.
  4. Open the order-events topic, select + Subscription, name it inventory, keep the defaults, and select Create.
  5. Select + Subscription again, name it customer-care, keep the defaults, and select Create.
  6. Set the namespace hostname copied in Lab 1 as SERVICEBUS_FQNS in every terminal used by this lab.

Option B: Azure CLI

az servicebus topic create --resource-group $resourceGroup --namespace-name $namespace --name order-events
  az servicebus topic subscription create --resource-group $resourceGroup --namespace-name $namespace --topic-name order-events --name inventory
  az servicebus topic subscription create --resource-group $resourceGroup --namespace-name $namespace --topic-name order-events --name customer-care
  $env:SERVICEBUS_FQNS = "$namespace.servicebus.windows.net"
Note: Complete either Option A or Option B, not both.

Checkpoint: The portal shows one topic and two subscriptions, each with zero active messages.

2. Create publisher and subscriber applications

New-Item -ItemType Directory -Path lab2-topic
  Set-Location lab2-topic
  mvn archetype:generate "-DgroupId=com.contoso.orders" "-DartifactId=order-publisher" "-DarchetypeArtifactId=maven-archetype-quickstart" "-DinteractiveMode=false"
  mvn archetype:generate "-DgroupId=com.contoso.events" "-DartifactId=event-subscriber" "-DarchetypeArtifactId=maven-archetype-quickstart" "-DinteractiveMode=false"

Configure both POM files exactly as in Lab 1: Java 17, azure-messaging-servicebus, azure-identity, and exec-maven-plugin.

3. Build the order publisher

Replace order-publisher/src/main/java/com/contoso/orders/App.java with:

package com.contoso.orders;
  import com.azure.identity.DefaultAzureCredentialBuilder;
  import com.azure.messaging.servicebus.*;
  import java.time.Instant;

  public class App {
    public static void main(String[] args) {
      String fqns = System.getenv("SERVICEBUS_FQNS");
      if (fqns == null) throw new IllegalStateException("Set SERVICEBUS_FQNS first.");
      try (ServiceBusSenderClient sender = new ServiceBusClientBuilder()
          .fullyQualifiedNamespace(fqns).credential(new DefaultAzureCredentialBuilder().build())
          .sender().topicName("order-events").buildClient()) {
        String orderId = "ORD-" + Instant.now().toEpochMilli();
        String body = "{\"orderId\":\"%s\",\"product\":\"AZ-BOOK\",\"quantity\":2}".formatted(orderId);
        sender.sendMessage(new ServiceBusMessage(body).setMessageId(orderId)
            .setSubject("OrderCreated").setContentType("application/json"));
        System.out.println("Published one event for " + orderId);
      }
    }
  }

4. Build the reusable subscriber

Replace event-subscriber/src/main/java/com/contoso/events/App.java with:

package com.contoso.events;
  import com.azure.identity.DefaultAzureCredentialBuilder;
  import com.azure.messaging.servicebus.*;
  import java.util.Scanner;

  public class App {
    public static void main(String[] args) {
      if (args.length != 1) throw new IllegalArgumentException("Pass inventory or customer-care.");
      String subscription = args[0];
      String fqns = System.getenv("SERVICEBUS_FQNS");
      if (fqns == null) throw new IllegalStateException("Set SERVICEBUS_FQNS first.");
      ServiceBusProcessorClient processor = new ServiceBusClientBuilder()
          .fullyQualifiedNamespace(fqns).credential(new DefaultAzureCredentialBuilder().build())
          .processor().topicName("order-events").subscriptionName(subscription).disableAutoComplete()
          .processMessage(context -> {
            System.out.printf("[%s] received %s: %s%n", subscription,
                context.getMessage().getMessageId(), context.getMessage().getBody().toString());
            context.complete();
          })
          .processError(context -> System.err.printf("[%s] %s%n", subscription, context.getException().getMessage()))
          .buildProcessorClient();
      processor.start();
      System.out.println("Listening on " + subscription + ". Press Enter to stop.");
      new Scanner(System.in).nextLine();
      processor.close();
    }
  }

5. Run three processes and verify fan-out

# Terminal 1: run the reusable subscriber for inventory
  Set-Location lab2-topic/event-subscriber
  mvn compile exec:java "-Dexec.mainClass=com.contoso.events.App" "-Dexec.args=inventory"

  # Terminal 2: run the same subscriber for customer care
  Set-Location lab2-topic/event-subscriber
  mvn compile exec:java "-Dexec.mainClass=com.contoso.events.App" "-Dexec.args=customer-care"

  # Terminal 3: publish one event
  Set-Location lab2-topic/order-publisher
  mvn compile exec:java "-Dexec.mainClass=com.contoso.orders.App"

Expected result: The publisher reports one event. Both subscriber terminals display the same MessageId and body. Each subscription returns to zero active messages.

Challenge: Stop only customer care and publish again. Inventory processes its copy immediately while customer care retains its copy until restarted.
