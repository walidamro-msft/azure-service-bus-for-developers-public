Lab 4 - Diagnose and recover poison messages

Create an invalid order, dead-letter it with diagnostics, repair it, and re-drive it safely.

60 minutesThree Java appsDead-letter queue

Case study: isolate an invalid order

A malformed order must not block valid work or disappear without evidence. The validator moves it to the queue's dead-letter subqueue with a reason and description. Recovery inspects that metadata, sends a corrected copy, and only then completes the original.

flowchart LR
 P[Poison producer] -- invalid order --> Q[orders queue]
 Q -- validation failure --> D[orders DLQ]
 D -- inspect and repair --> R[Recovery app]
 R -- corrected copy --> Q
Important: Service Bus does not automatically clean the DLQ. Alert on dead-letter counts and retain reason, description, and correlation IDs.

2. Send an invalid order

Replace the poison producer's App.java with:

package com.contoso.poison;
import com.azure.identity.DefaultAzureCredentialBuilder;
import com.azure.messaging.servicebus.*;
import java.time.Instant;

public class App {
  public static void main(String[] args) {
    String fqns = System.getenv("SERVICEBUS_FQNS");
    try (ServiceBusSenderClient sender = new ServiceBusClientBuilder()
        .fullyQualifiedNamespace(fqns).credential(new DefaultAzureCredentialBuilder().build())
        .sender().queueName("orders").buildClient()) {
      sender.sendMessage(new ServiceBusMessage("not-json")
          .setMessageId("POISON-" + Instant.now().toEpochMilli()).setSubject("OrderCreated"));
      System.out.println("Sent one invalid order.");
    }
  }
}

3. Validate and dead-letter

Replace the validator's App.java with:

package com.contoso.validation;
import com.azure.identity.DefaultAzureCredentialBuilder;
import com.azure.messaging.servicebus.*;
import com.azure.messaging.servicebus.models.DeadLetterOptions;
import com.fasterxml.jackson.databind.*;
import java.time.Duration;

public class App {
  public static void main(String[] args) {
    String fqns = System.getenv("SERVICEBUS_FQNS");
    try (ServiceBusReceiverClient receiver = new ServiceBusClientBuilder()
        .fullyQualifiedNamespace(fqns).credential(new DefaultAzureCredentialBuilder().build())
        .receiver().queueName("orders").buildClient()) {
      ServiceBusReceivedMessage message = receiver.receiveMessage(Duration.ofSeconds(20));
      if (message == null) throw new IllegalStateException("No order was available.");
      try {
        JsonNode order = new ObjectMapper().readTree(message.getBody().toBytes());
        if (!order.hasNonNull("orderId")) throw new IllegalArgumentException("orderId is required");
        receiver.complete(message);
        System.out.println("Validated and completed " + order.get("orderId").asText());
      } catch (Exception exception) {
        receiver.deadLetter(message, new DeadLetterOptions().setDeadLetterReason("InvalidOrderJson")
            .setDeadLetterErrorDescription(exception.getMessage()));
        System.out.println("Dead-lettered " + message.getMessageId() + ": InvalidOrderJson");
      }
    }
  }
}

4. Inspect, repair, and re-drive

Replace the recovery app's App.java with:

package com.contoso.recovery;
import com.azure.identity.DefaultAzureCredentialBuilder;
import com.azure.messaging.servicebus.*;
import com.azure.messaging.servicebus.models.SubQueue;
import java.time.Duration;

public class App {
  public static void main(String[] args) {
    String fqns = System.getenv("SERVICEBUS_FQNS");
    ServiceBusClientBuilder builder = new ServiceBusClientBuilder().fullyQualifiedNamespace(fqns)
        .credential(new DefaultAzureCredentialBuilder().build());
    try (ServiceBusReceiverClient dlq = builder.receiver().queueName("orders")
             .subQueue(SubQueue.DEAD_LETTER_QUEUE).buildClient();
         ServiceBusSenderClient sender = builder.sender().queueName("orders").buildClient()) {
      ServiceBusReceivedMessage failed = dlq.receiveMessage(Duration.ofSeconds(20));
      if (failed == null) throw new IllegalStateException("The DLQ is empty.");
      System.out.println("Reason: " + failed.getDeadLetterReason());
      System.out.println("Description: " + failed.getDeadLetterErrorDescription());
      ServiceBusMessage repaired = new ServiceBusMessage("{\"orderId\":\"ORD-RECOVERED\",\"product\":\"AZ-BOOK\",\"quantity\":1}")
          .setMessageId(failed.getMessageId() + "-repaired").setCorrelationId(failed.getMessageId())
          .setSubject(failed.getSubject()).setContentType("application/json");
      sender.sendMessage(repaired);
      dlq.complete(failed);
      System.out.println("Re-drove the repaired copy and completed the DLQ original.");
    }
  }
}

5. Run the recovery drill

mvn -f lab4-dlq/poison-producer/pom.xml compile exec:java "-Dexec.mainClass=com.contoso.poison.App"
mvn -f lab4-dlq/order-validator/pom.xml compile exec:java "-Dexec.mainClass=com.contoso.validation.App"
# Confirm the portal shows one dead-lettered message.
mvn -f lab4-dlq/dlq-recovery/pom.xml compile exec:java "-Dexec.mainClass=com.contoso.recovery.App"
# Consume the repaired copy.
mvn -f lab4-dlq/order-validator/pom.xml compile exec:java "-Dexec.mainClass=com.contoso.validation.App"

Expected result: The DLQ count rises to one after validation, then returns to zero after recovery. The final validator run completes ORD-RECOVERED, and the active queue count returns to zero.

Challenge: Abandon messages with unknown dead-letter reasons for investigation instead of automatically re-driving them.
