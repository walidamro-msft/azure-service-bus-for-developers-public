Lab 1 - Queue-based order intake

Create a Service Bus queue, then build separate Java producer and receiver applications.

60 minutesTwo Java appsQueues

Case study: protect checkout from traffic spikes

Contoso Shop must accept orders even when fulfilment is busy or offline. The checkout producer writes each order to a durable queue, while a separate receiver drains the queue at its own pace.

flowchart LR
  C[Java checkout producer] -- OrderCreated --> Q[orders queue]
  Q -- competing consumer --> F[Java fulfilment receiver]

Outcome: Deploy the queue, build two independently runnable Maven applications, send ten orders, and prove that the receiver settles every message.

1. Provision the queue

Use Standard so the same namespace can support topics and sessions in later labs. Local authentication is disabled; your Azure identity receives data-plane access through RBAC.

Option A: Azure portal

  1. Open the Create a Service Bus namespace page.
  2. Select your subscription, create or select rg-sb-workshop, enter a globally unique namespace such as sb-contoso-<unique>, choose a region, and select the Standard pricing tier.
  3. Select Review + create, then Create. When deployment finishes, open the namespace.
  4. Under Settings, open Shared access policies and disable Local Authentication. The applications use Microsoft Entra ID instead of access keys.
  5. Under Entities, select Queues and + Queue. Set the name to orders, set Max delivery count to 5, keep the other defaults, and select Create.
  6. Open Access control (IAM), select Add > Add role assignment, choose Azure Service Bus Data Owner, and assign it to your signed-in user.
  7. On the namespace Overview page, copy the hostname ending in .servicebus.windows.net. Set that value as SERVICEBUS_FQNS in every terminal used by this lab.

Option B: Azure CLI

az login
$suffix = Get-Random -Maximum 99999
$resourceGroup = "rg-sb-workshop"
$namespace = "sb-contoso-$suffix"
az group create --name $resourceGroup --location eastus
az servicebus namespace create --resource-group $resourceGroup --name $namespace --location eastus --sku Standard --disable-local-auth true
az servicebus queue create --resource-group $resourceGroup --namespace-name $namespace --name orders --max-delivery-count 5
$namespaceId = az servicebus namespace show --resource-group $resourceGroup --name $namespace --query id -o tsv
$userId = az ad signed-in-user show --query id -o tsv
az role assignment create --assignee $userId --role "Azure Service Bus Data Owner" --scope $namespaceId
$env:SERVICEBUS_FQNS = "$namespace.servicebus.windows.net"
Note: Complete either Option A or Option B, not both. RBAC may take several minutes to propagate. Both applications read SERVICEBUS_FQNS from the environment.

2. Create two Maven applications

New-Item -ItemType Directory -Path lab1-queue
Set-Location lab1-queue
mvn archetype:generate "-DgroupId=com.contoso.checkout" "-DartifactId=checkout-producer" "-DarchetypeArtifactId=maven-archetype-quickstart" "-DinteractiveMode=false"
mvn archetype:generate "-DgroupId=com.contoso.fulfilment" "-DartifactId=fulfilment-receiver" "-DarchetypeArtifactId=maven-archetype-quickstart" "-DinteractiveMode=false"

3. Build the checkout producer

Replace checkout-producer/src/main/java/com/contoso/checkout/App.java with:

package com.contoso.checkout;

import com.azure.identity.DefaultAzureCredentialBuilder;
import com.azure.messaging.servicebus.ServiceBusClientBuilder;
import com.azure.messaging.servicebus.ServiceBusMessage;
import com.azure.messaging.servicebus.ServiceBusSenderClient;

public class App {
    public static void main(String[] args) {
        String fqns = requireEnvironment("SERVICEBUS_FQNS");
        try (ServiceBusSenderClient sender = new ServiceBusClientBuilder()
                .fullyQualifiedNamespace(fqns)
                .credential(new DefaultAzureCredentialBuilder().build())
                .sender().queueName("orders").buildClient()) {
            for (int number = 1; number <= 10; number++) {
                String orderId = "ORD-%03d".formatted(number);
                String body = "{\"orderId\":\"%s\",\"product\":\"AZ-BOOK\",\"quantity\":%d}".formatted(orderId, number);
                sender.sendMessage(new ServiceBusMessage(body).setMessageId(orderId)
                        .setSubject("OrderCreated").setContentType("application/json"));
                System.out.println("Sent " + orderId);
            }
        }
    }
    private static String requireEnvironment(String name) {
        String value = System.getenv(name);
        if (value == null || value.isBlank()) throw new IllegalStateException("Set " + name + " first.");
        return value;
    }
}

4. Build the fulfilment receiver

Replace fulfilment-receiver/src/main/java/com/contoso/fulfilment/App.java with:

package com.contoso.fulfilment;

import com.azure.identity.DefaultAzureCredentialBuilder;
import com.azure.messaging.servicebus.ServiceBusClientBuilder;
import com.azure.messaging.servicebus.ServiceBusProcessorClient;
import com.azure.messaging.servicebus.models.ServiceBusReceiveMode;
import java.util.Scanner;

public class App {
    public static void main(String[] args) {
        String fqns = requireEnvironment("SERVICEBUS_FQNS");
        ServiceBusProcessorClient processor = new ServiceBusClientBuilder()
                .fullyQualifiedNamespace(fqns)
                .credential(new DefaultAzureCredentialBuilder().build())
                .processor().queueName("orders")
                .receiveMode(ServiceBusReceiveMode.PEEK_LOCK).disableAutoComplete()
                .processMessage(context -> {
                    System.out.printf("Received %s (delivery %d)%n", context.getMessage().getMessageId(), context.getMessage().getDeliveryCount());
                    context.complete();
                })
                .processError(context -> System.err.println(context.getException().getMessage()))
                .buildProcessorClient();
        processor.start();
        System.out.println("Receiver is running. Press Enter to stop.");
        new Scanner(System.in).nextLine();
        processor.close();
    }
    private static String requireEnvironment(String name) {
        String value = System.getenv(name);
        if (value == null || value.isBlank()) throw new IllegalStateException("Set " + name + " first.");
        return value;
    }
}

5. Run and verify

Open two PowerShell terminals, set the same SERVICEBUS_FQNS value in each, and run:

# Terminal 1
Set-Location lab1-queue/fulfilment-receiver
mvn compile exec:java "-Dexec.mainClass=com.contoso.fulfilment.App"

# Terminal 2
Set-Location lab1-queue/checkout-producer
mvn compile exec:java "-Dexec.mainClass=com.contoso.checkout.App"

Expected result: The producer prints ten sent order IDs. The receiver prints those ten IDs and completes them. The queue's active message count returns to zero.

Challenge: Stop the receiver, send another batch, and observe durable buffering. Run two receiver instances and confirm each order is processed by only one competing consumer.
