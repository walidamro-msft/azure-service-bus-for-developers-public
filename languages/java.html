Scenario and lab map

Each lab is a complete case study: provision the required messaging entities, build independently runnable applications, execute the scenario, and verify its observable result.

  1. Queue-based order intake - create a queue plus separate Java producer and receiver apps
  2. Publish one event to two subscribers - create a topic plus publisher and two subscriber processes
  3. Session-ordered fulfilment - create a session queue plus workflow sender and processor
  4. Dead-letter recovery - produce, reject, inspect, repair, and re-drive an invalid order
  5. Serverless load-to-storage integration - provision, deploy, load-test, and verify the complete cloud path
Authentication: Sign in with az login. Local code uses DefaultAzureCredential; Azure-hosted components use managed identity. Do not create or copy a Service Bus connection string.
