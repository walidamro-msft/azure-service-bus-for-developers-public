Working with queues in JavaScript / TypeScript

Map Service Bus processing decisions to receiver creation options, callback subscriptions, pull receive methods, settlement calls, sessions, and client lifetimes.

Node.js@azure/service-busReceiver subscription and pull models

Package, authentication, and client

Install @azure/service-bus and @azure/identity. Use DefaultAzureCredential locally and managed identity for Azure-hosted workloads. Grant producers Azure Service Bus Data Sender and consumers Azure Service Bus Data Receiver.

npm install @azure/service-bus @azure/identity
import { ServiceBusClient } from "@azure/service-bus";
import { DefaultAzureCredential } from "@azure/identity";

const fqns = process.env.SERVICEBUS_FQNS;
if (!fqns) throw new Error("Set SERVICEBUS_FQNS.");

const client = new ServiceBusClient(fqns, new DefaultAzureCredential());
Production identity: the same code can select a workload or managed identity in Azure. Do not embed connection strings or access keys.

Send to a queue

const sender = client.createSender("orders-processing");

await sender.sendMessages({
  body: { orderId: "ORD-123", status: "created" },
  messageId: crypto.randomUUID(),
  correlationId: "order:ORD-123",
  subject: "OrderCreated.v1",
  contentType: "application/json",
  applicationProperties: { schemaVersion: 1 },
  timeToLive: 30 * 60 * 1000
});

timeToLive is milliseconds. Reuse the sender and close it during shutdown. createMessageBatch and tryAddMessage provide size-aware batching.

Continuous processing with receiver.subscribe()

This SDK does not expose a separate processor class. A ServiceBusReceiver owns the receive link, and subscribe starts callback-driven processing on that receiver.

const receiver = client.createReceiver("orders", {
  receiveMode: "peekLock",
  maxAutoLockRenewalDurationInMs: 5 * 60 * 1000,
  identifier: `orders-worker:${process.env.COMPUTERNAME ?? process.pid}`
});

const subscription = receiver.subscribe({
  processMessage: async message => {
    try {
      await processOrder(message.body);
      await receiver.completeMessage(message);
    } catch (error) {
      if (error instanceof PermanentOrderError) {
        await receiver.deadLetterMessage(message, {
          deadLetterReason: "InvalidOrder",
          deadLetterErrorDescription: error.message
        });
      } else {
        await receiver.abandonMessage(message);
      }
    }
  },
  processError: async args => {
    console.error(args.errorSource, args.entityPath, args.error);
  }
}, {
  autoCompleteMessages: false,
  maxConcurrentCalls: 4
});

await shutdownSignal;
await subscription.close();
await receiver.close();
await client.close();

Keep the subscription handle so shutdown can stop callback delivery. The error callback reports receive and handler infrastructure failures; it does not replace explicit settlement in processMessage.

Receiver and subscription option mapping

JavaScript/TypeScript splits common controls between options passed to createReceiver and options passed to subscribe.

autoCompleteMessages

Configure onsubscribe
Defaulttrue
Behavior

Completes after a successful callback and abandons when the callback throws, unless the callback already settled the message.

Strategy

Set false for explicit complete, abandon, defer, or dead-letter decisions. It is ignored in receiveAndDelete mode.

maxConcurrentCalls

Configure onsubscribe
Default1
Behavior

Limits simultaneous calls to processMessage for this subscription.

Strategy

Potential concurrency is replicas multiplied by this value. Increase while event-loop delay, memory, downstream pools, and lock completion remain healthy.

maxAutoLockRenewalDurationInMs

Configure oncreateReceiver
Default300000 ms
Behavior

Sets the total automatic renewal window while a PeekLock message is processed. Renewal stops after settlement.

Strategy

Set a finite value above expected handler time. Set zero to disable automatic renewal.

Prefetch

Configure onNot exposed
DefaultSDK-managed receive credit
Behavior

The current @azure/service-bus receiver options do not expose a public prefetch-count setting.

Strategy

Tune throughput with maxConcurrentCalls, handler efficiency, batch receive size, and replica count. Do not invent a prefetchCount option.

receiveMode

Configure oncreateReceiver
DefaultpeekLock
Behavior

Accepts "peekLock" or "receiveAndDelete".

Strategy

Use peekLock for business work. Use receiveAndDelete only when loss after transfer is acceptable.

subQueueType

Configure oncreateReceiver
DefaultMain entity
Behavior

Accepts "deadLetter" or "transferDeadLetter" to target a subqueue.

Strategy

Create a separate receiver for deliberate DLQ recovery. Do not attach the normal worker to a subqueue.

identifier

Configure oncreateReceiver
DefaultGenerated unique value
Behavior

Names the receiver in selected logs and exceptions without affecting routing.

Strategy

Use a role plus unique replica suffix for operational correlation.

Pull and iterative receiving

Use receiveMessages for bounded batches or getMessageIterator for an application-owned asynchronous loop.

const receiver = client.createReceiver("orders", {
  receiveMode: "peekLock",
  identifier: `orders-batch:${process.pid}`
});

try {
  const messages = await receiver.receiveMessages(20, { maxWaitTimeInMs: 5000 });
  for (const message of messages) {
    try {
      await processOrder(message.body);
      await receiver.completeMessage(message);
    } catch {
      await receiver.abandonMessage(message);
    }
  }
} finally {
  await receiver.close();
  await client.close();
}

The returned array may contain fewer messages than requested. Pull receivers also expose peekMessages, receiveDeferredMessages, and renewMessageLock.

Sessions, subscriptions, and dead-letter queues

  • For a topic subscription, call createReceiver(topicName, subscriptionName, options).
  • For one session, call acceptSession; for the next available session, call acceptNextSession. Your application manages concurrency across accepted session receivers.
  • For dead-letter recovery, call createReceiver(entity, { subQueueType: "deadLetter", receiveMode: "peekLock" }).
  • Use an AbortSignal with receive or subscribe options when cancellation must be coordinated with the rest of the Node.js application.
