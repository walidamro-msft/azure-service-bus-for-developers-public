Introductions

Azure Service Bus is a fully managed enterprise message broker that enables reliable asynchronous communication between applications and services. This module focuses on when and why to use Service Bus in distributed systems.

Overview

  • Business messaging: transfer orders, inventory updates, journals, and other business events.
  • Application decoupling: producers and consumers do not need to be online at the same time.
  • Load leveling: smooth traffic spikes so downstream services are not overwhelmed.
  • Competing consumers: safely distribute work across multiple workers.

Concepts

Queue

A queue supports point-to-point messaging where each message is processed by one successful consumer. Messages are durably stored until they are received and settled.

EntityPattern
Queue Point-to-point messaging between sender and receiver.
Topic Publish-subscribe messaging with one sender and multiple subscribers.

Queue

Topics

Topics support publish-subscribe messaging. Multiple subscriptions can independently receive copies of messages, and rules can filter which messages each subscription gets.

Topic

Namespaces

A namespace is the top-level container for messaging entities such as queues and topics. It represents your Service Bus capacity and management boundary.

Advanced features

Message sessions

Enable ordered processing and correlation patterns such as request-response for related message streams.

Autoforwarding

Automatically chains a queue or subscription to another entity in the same namespace for staged routing workflows.

Dead-lettering

Uses a dead-letter subqueue for messages that cannot be delivered or processed, enabling diagnostics and replay workflows.

Scheduled delivery

Schedules messages for future availability so consumers process them at a specific time.

Message Deferral

Defers a message when processing cannot happen immediately, allowing retrieval later without message loss.

Transactions

Groups operations into an atomic unit, supporting reliable once-only style processing patterns.

Filters and Actions

Applies subscription rules to select and optionally annotate topic messages per subscriber needs.

Auto-delete on idle

Automatically deletes idle entities after a configured interval to reduce unused resource sprawl.

Duplicate detection

Prevents duplicate message processing by discarding repeated message sends within the detection window.

Batch deletion of messages

Removes expired or no-longer-relevant messages in bulk from queues or subscriptions.

Security

Supports shared access signatures (SAS), Azure RBAC, and managed identities, with AMQP and HTTP/REST access models.

Geo-Replication

Supports continuity scenarios by enabling data processing in alternate regions during regional disruptions.

Compliance with standards and protocols

Uses AMQP 1.0 as the primary wire protocol and supports JMS compatibility in supported tiers.
