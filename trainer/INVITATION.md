## Azure Service Bus Developer Workshop

You are invited to a hands-on Azure Service Bus workshop focused on designing, building, securing, and operating reliable asynchronous messaging solutions.

### What we will cover

- Azure Service Bus fundamentals: namespaces, queues, topics, subscriptions, messages, and settlement.
- Messaging architecture decisions across Service Bus, Event Grid, Event Hubs, Queue Storage, and synchronous APIs.
- Service Bus tiers, partitioning, Premium messaging units, availability, and cost considerations.
- Creating and managing namespaces, queues, topics, and subscriptions with the Azure portal, Azure CLI, and Bicep.
- Queue processing, publish/subscribe, filters, retries, idempotency, and dead-letter handling.
- Advanced messaging patterns with sessions, duplicate detection, scheduling, deferral, and message expiration.
- Passwordless application development using .NET, Java, JavaScript/TypeScript, or Python.
- Security and governance with managed identities, RBAC, private networking, and policy controls.
- Monitoring and troubleshooting with metrics, logs, alerts, backlog analysis, and operational runbooks.

### Who should attend

- Cloud architects, integration developers, application developers, platform engineers, and DevOps engineers.

### Prerequisites

- Basic Azure knowledge.
- Familiarity with APIs, events, or asynchronous application concepts.
- Azure subscription access for hands-on labs (recommended).
- Visual Studio Code, Azure CLI, Git, and one supported language runtime for the selected lab track.

### Outcome

By the end of this workshop, attendees will be able to choose an appropriate Azure messaging architecture, design Service Bus entities and message contracts, build passwordless producers and consumers, apply reliable processing patterns, and operate secure and observable messaging workloads.
