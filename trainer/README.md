# Azure Service Bus Workshop Trainer Package

Use these files to prepare, invite attendees, and deliver the workshop.

## Contents

- [Trainer Hub](index.html) - browser-ready facilitator workspace with the copyable invite, delivery flow, shortcuts, and guidance summary.
- [Invitation](INVITATION.md) - concise attendee-facing workshop description that can be copied into an email, calendar invitation, or registration page.
- [Trainer notes](TRAINER-NOTES.md) - preparation checklist, delivery plan, module cues, demonstrations, checkpoints, troubleshooting, and cleanup.
- [Workshop home](../index.html) - the attendee learning path and language tracks.

## Recommended Format

The complete workshop works best as two instructor-led days:

- **Day 1:** messaging foundations, architecture, tier selection, deployment, queues, topics, and subscriptions.
- **Day 2:** sessions, dead-letter recovery, security, monitoring, administration, and the optional serverless integration lab.

For a one-day delivery, select the architecture modules and core queue/topic exercises while keeping the invitation focused on workshop outcomes rather than scheduling details.

## Before Publishing an Invitation

1. Select one language track for the live exercises: .NET, Java, JavaScript/TypeScript, or Python.
2. State whether attendees use personal Azure subscriptions, a provided sandbox, or shared resources.
3. Confirm whether the optional serverless integration lab is in scope.
4. Send the prerequisites early enough for attendees to verify local tools and Azure access.
