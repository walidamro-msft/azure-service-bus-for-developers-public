Deploy with Azure CLI and Bicep

Move from click-ops to repeatable infrastructure by codifying Service Bus namespaces and entities.

Hands-on~30 minModule 5 of 11

CLI baseline flow

az group create --name rg-sb-workshop --location eastus

az servicebus namespace create \
  --resource-group rg-sb-workshop \
  --name sb-workshop-demo \
  --sku Standard

az servicebus queue create \
  --resource-group rg-sb-workshop \
  --namespace-name sb-workshop-demo \
  --name orders-queue

Bicep template pattern

param location string = resourceGroup().location
param namespaceName string

resource sb 'Microsoft.ServiceBus/namespaces@2022-10-01-preview' = {
  name: namespaceName
  location: location
  sku: {
    name: 'Standard'
    tier: 'Standard'
  }
}

resource queue 'Microsoft.ServiceBus/namespaces/queues@2022-10-01-preview' = {
  name: '${sb.name}/orders-queue'
  properties: {
    maxDeliveryCount: 10
    requiresDuplicateDetection: true
  }
}

Deploy template

az deployment group create \
  --resource-group rg-sb-workshop \
  --template-file main.bicep \
  --parameters namespaceName=sb-workshop-demo

Use one template per environment with parameter files for naming, SKU, and policy options.

IaC practice checklist

  • Keep naming and tags parameterized.
  • Enforce diagnostics settings as code where possible.
  • Run what-if before deployment in shared environments.
  • Use CI/CD to validate and apply Bicep templates.
