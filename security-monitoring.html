Security and Monitoring

Harden access and make operations observable with managed identity, RBAC, diagnostics, and actionable alerting.

Hands-on~45 minModule 9 of 11

Identity and RBAC

  • Use DefaultAzureCredential with managed identity for hosted workloads.
  • Assign least-privilege roles: Data Sender, Data Receiver, or Data Owner only when required.
  • Avoid shared keys for production app paths.

Network security

Limit public access, prefer private endpoints, and use firewall/network rules to constrain ingress patterns.

Diagnostics and metrics

SignalWhy it matters
ActiveMessages / DeadletteredMessagesBacklog growth and poison message accumulation
IncomingRequests / ThrottledRequestsCapacity pressure and throughput issues
ServerErrors / UserErrorsAvailability and client correctness problems

Alert strategy

  1. Alert on sustained DLQ growth over threshold.
  2. Alert on throttles and server errors.
  3. Alert on unusual backlog age and processing latency.
