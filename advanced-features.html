Sessions for ordered workflows

Sessions bind related messages to the same ordered stream. Use SessionId for workflows like order processing where sequence correctness matters.

Pattern: set SessionId to aggregate root key (for example orderId).

Scheduling and deferral

var delayed = new ServiceBusMessage("follow-up")
{
    ScheduledEnqueueTime = DateTimeOffset.UtcNow.AddMinutes(15)
};
await sender.SendMessageAsync(delayed);

// Deferral keeps message hidden until sequence retrieval
await args.DeferMessageAsync(args.Message);

Duplicate detection

Enable duplicate detection on queue/topic and always send deterministic MessageId values from upstream systems to prevent replay side effects.
