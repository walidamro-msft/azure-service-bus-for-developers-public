Deploy via Azure Portal

Provision Service Bus resources interactively and validate connectivity before moving to infrastructure as code.

Hands-on~25 minModule 4 of 11

Portal setup checklist

  1. Create or select a resource group for workshop assets.
  2. Create a Service Bus namespace and choose the target tier.
  3. Create at least one queue and one topic with a subscription.
  4. Capture the fully qualified namespace for SDK configuration.

Recommended namespace configuration

  • Use naming convention: sb-<workload>-<env>-<region>.
  • Enable local auth only for learning; prefer managed identity in production.
  • Apply tags: Environment, Owner, Application, CostCenter.
  • Keep public network access limited and plan private endpoints for real workloads.

Validation and smoke test

Use Service Bus Explorer in the portal to send a test message and verify it appears in the target queue/subscription. This confirms namespace access and entity readiness before coding.

Checkpoint: Successfully send and receive one message from queue and subscription paths.
